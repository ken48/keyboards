0. Как я пришел к необычным клавиатурам?

	0.0. Изначально я озадачился вопросом эргономики и стиля рабочего места. После стола, монитора и кресла внимание перешло на клавиатуру и мышь.

	0.1. С мышью все понятно: выбирай точную и удобную. А вот с клавиатурами - они все казались одинаковами, различались только по внешности. И я начал искать идеальную по эстетическим соображениям клавиатуру: круто выглядящую, с приятным филингом (тактильные ощущения при печати, приятный звук, качество материалов).

	0.2. Потом пришло понимание, что можно использовать функциональные возможности таких клавиатур. Точки роста были замечены такие:
		- точность попадания на клавиши, решается через выбор профиля кейкапов
		- общая площадь обычных полноразмерных клавиатур казалась избыточной, т.к. надо двигать руки с домашней позиции, мышь казалась где-то далеко в стороне. Поэтому хотелось компактных раскладок (75, 65 или даже 60), которые можно и нужно было донастраивать под себя.

	0.3. В целом, можно считать, что за год экспериментов я определился с оптимальным набором характеристик:
		- по свичам это самые легкие из тактильных и обязательно материал стема нейлон для более мягкого звука
		- профиль кейкапов скульптурный и высокий (KSA или MT3)
		- из-за общей высоты обязательна подставка под запястья (материал - только дерево)

		По раскладке - для программиста это 75%. Потому что нужны стрелки для работы с текстом и F-ряд для стандартных хоткеев в файловом менеджере и средах разработки.

3. Проблемы в кейсах 

	3.1. общие для русского и английского набора, вне зависимости от того, код это или natural language текст:
		- выбор конкретного языка ввода, а не переключение между ними
		- знаки препинания заставляют использовать шифт на цифровом ряду, хотя цифры используются значительно реже
		- при наборе цифр их в ряду набирать неудобно
		- использование backspace из положения рук "на домашнем ряду"
		- шифт на мизинце заставляет делать растяжку для набора большой буквы (как правило, одиночной)
		- из-за быстрого набора часто заглавной становится не только первая, но и вторая буква в словах

	3.2. специфические для русского:
		- при наборе текста из нескольких предложений точка и запятая неудобно набирается в цифровом ряду. Особенно запятая

4. Решение для стандартных клавиатур:

	Перенастройка и донастройка через middleware на уровне ОС (mac):
		- выбор конкретного языка на нижнем ряду (на крайних клавишах: fn - En, правый option - Ru)
		- замена дефолтных значений цифрового ряда с цифр на символы (цифры получаются с шифтом)
		- замена Caps Lock на fn, активирующий медиа-слой (F-row начинает регулировать яркость, звук и тд)
		- "home row numpad" расположение цифр в районе home row столбиком как на нампаде (доработанный порядок учитывает частоту использования цифр - на домашнем ряду 0123) 
		- при быстром наборе ошибки замечаются не сразу, поэтому есть очень удобный прием удалять по слову целиком. Стандартный option + backspace заставляет уносить руки далеко от домашнего ряда. Вместо этого проще правой рукой нажать правый шифт + пробел.
		- для решения проблемы запятой на русском я использую комбинацию левый шифт + пробел. Это добавляет русскую запятую и сразу ставит пробел.
		- для решения проблемы точки на русском я использую комбинацию Caps Lock + пробел. Это добавляет русскую точку, сразу ставит пробел и активирует однократный шифт для начала следующего предложения с заглавной буквы.

		Обязательно в системных настройках клавиатуры надо отключить смену языка при однократном нажатии на fn, а также замену двойного пробела на точку.

	 Перенастройка и донастройка через VIA для QMK клавиатур:
	 	- для переключения языков в middleware на это определены F18 и F19, через механику ModTap они назначены на левый и правый контролы соответственно (крайние клавиши нижнего ряда, не считая стрелок)

	  Проблемы с шифтами пока не решил. Попробовал sticky modifier на левом шифте, но по ходу эксплуатации появились побочные эффекты, в целом, сводящие это удобство на нет.
